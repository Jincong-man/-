# security-audit

> 锦聪的小店 — 代码安全审计技能

## 功能概述

这个技能对项目代码进行安全审计，从以下四个维度检查：

| 检查项 | 说明 | 触发词 |
|--------|------|--------|
| **敏感信息泄露** | 密码、密钥、Token、邮箱、手机号等是否硬编码在代码中 | "查敏感信息"、"有没有密码泄露" |
| **注入漏洞** | SQL/HTML/JS 注入、XSS 跨站脚本攻击风险 | "查注入漏洞"、"有没有XSS风险" |
| **明文配置** | 配置文件中的密码、API 密钥是否明文存储 | "配置文件安全"、"明文密码" |
| **综合审计** | 以上全部 + 其他安全隐患（存储安全、传输安全、认证安全） | "安全审计"、"安全检查" |

---

## 检查工作的流程

对项目源文件逐项审计，按以下四个维度执行：

---

### 一、敏感信息硬编码检查 🔴

**解释**：硬编码（Hardcode）指的是把密码、密钥等敏感数据直接写在代码里，而不是放在外部配置文件或环境变量中。一旦代码被分享或上传到 GitHub，这些秘密就暴露了。

检查以下模式：

| 检查项 | 搜索模式 | 风险等级 |
|--------|----------|----------|
| **硬编码密码** | `password === '...'`、`password == "..."`、`pwd:`、`passwd:` | 🔴 严重 |
| **API 密钥/Token** | `apiKey`、`api_key`、`token`、`secret`、`access_key` 后跟字符串 | 🔴 严重 |
| **邮箱地址** | `@xxx.com` 格式的字符串 | 🟡 中等 |
| **手机号** | 连续 11 位数字 | 🟡 中等 |
| **数据库连接串** | `mongodb://`、`mysql://`、`postgres://` 含用户名密码的 URI | 🔴 严重 |
| **私钥/PEM** | `-----BEGIN`、`PRIVATE KEY` | 🔴 严重 |
| **内网地址/IP** | `192.168.`、`10.0.`、`172.16.` 等内网 IP | 🟡 中等 |

---

### 二、注入漏洞检查 🔴

**XSS（跨站脚本攻击）**：攻击者在网页中注入恶意脚本。例如：用户在搜索框输入 `<script>偷走你的密码</script>`，如果代码直接把这串内容插入到页面 HTML 中，脚本就会执行。

| 检查项 | 说明 | 危害 |
|--------|------|------|
| **未转义的用户输入** | `innerHTML`、`document.write()`、`eval()` 直接使用用户输入 | 🔴 恶意脚本可窃取用户数据 |
| **v-html 指令** | Vue 中直接渲染 HTML 字符串，跳过转义 | 🔴 同上 |
| **URL 参数直接拼接** | `location.href` / `location.search` 未校验直接使用 | 🟡 跳转劫持 |
| **setTimeout/setInterval 字符串** | 第一个参数为字符串而非函数 | 🟡 可能执行恶意代码 |
| **动态创建脚本** | `createElement('script')` + 动态内容 | 🔴 远程代码执行 |

**注入例子图解**（帮助理解什么是注入）：

```
用户输入（恶意）                  代码（不当处理）                      结果
─────────────────    ─────────────────────────    ─────────────────
<script>alert('XSS')</script>  →  innerHTML = 用户输入  →  ⚡ 脚本被执行！
'); DROP TABLE users; --        →  SQL = "SELECT * FROM users WHERE name='" + 输入 + "'"  →  ⚡ 数据库被删除！
```

---

### 三、配置文件明文存储检查 🟡

**解释**：配置文件（如 `.json`、`.env`、`.yaml` 等）如果包含密码或密钥，且以明文（未加密的原始文本）存储，任何能访问这个文件的人都能看到敏感信息。

检查以下文件：

| 文件 | 检查重点 |
|------|----------|
| `package.json` | 是否包含 token、密钥等字段 |
| `.vscode/settings.json` | 是否包含个人路径、敏感配置 |
| `.claude/settings.local.json` | 是否包含 token、个人路径 |
| `*.config.js` | 配置文件中是否硬编码了密码 |
| `.env` / `.env.local` | 是否存在且未被 `.gitignore` 排除 |

---

### 四、其他安全隐患 🟢

这些不是直接的漏洞，但会让系统更脆弱，属于"加固"范畴：

#### 存储安全

| 检查项 | 说明 | 风险 |
|--------|------|------|
| **localStorage 存密码** | 用户密码明文存入浏览器本地存储 | 🔴 任何人打开浏览器开发者工具就能看到 |
| **localStorage 存敏感数据** | Token、个人信息存入 localStorage | 🟡 XSS 攻击可读取 localStorage |
| **sessionStorage 存敏感数据** | 同上，作用域不同但风险类似 | 🟡 较 localStorage 稍好 |

#### 认证安全

| 检查项 | 说明 | 风险 |
|--------|------|------|
| **弱密码规则** | 注册时密码最小长度 < 8 位、没有复杂度要求 | 🟡 容易被暴力破解 |
| **无登录限制** | 无失败次数限制、无验证码 | 🟡 可被自动脚本暴力猜解 |
| **默认账号** | 代码中存在写死的默认账号密码 | 🔴 所有看代码的人都能登录 |
| **无会话过期** | 登录状态永久有效，没有超时机制 | 🟡 离开电脑后他人可操作 |

#### 传输安全

| 检查项 | 说明 | 风险 |
|--------|------|------|
| **HTTP 明文传输** | 密码通过 HTTP（非 HTTPS）传输 | 🔴 中间人可截获密码 |
| **密码在前端校验** | 密码逻辑完全在前端，后端无校验 | 🔴 绕过前端即可绕过所有校验 |

#### 依赖安全

| 检查项 | 说明 |
|--------|------|
| **过时依赖** | `package.json` 中依赖版本是否过旧、是否有已知漏洞 |
| **不必要的依赖** | 是否有引入但未使用的包 |

## 工作约定

1. **不修改代码**：只输出报告和建议，不直接改动源文件
2. **解释风险**：每条问题不只是说"有问题"，要解释"为什么这是问题"和"攻击者会怎么做"，适合新手理解
3. **给出可操作的修复方案**：每条建议附带具体的代码修改示例
4. **严重优先**：先报严重风险（🔴），再报中等（🟡），最后是加固建议（🟢）
5. **交叉引用已有的记忆**：说明问题时，关联到用户已有的知识背景
6. **排除无关文件**：`node_modules/`、`.git/`、`00.img/` 目录和图片文件不纳入检查
